직전 사이클(`implement_20260523_203219.md`)에서 workspace-scoped SSE + dead-letter 단건 redelivery까지 잡았다. 본 plan은 그 다음 자연 후속 두 가지.

## 개발 목적

1. 실행 중인 run의 stdout을 운영자가 실시간으로 볼 수 있게 한다. 현재는 종료된 후에만 댓글에 박힌 cleaned text나 로그 다운로드로만 접근 가능.
2. webhook receiver 장애 후 dead-letter가 여러 건 쌓였을 때 하나하나 누르지 않고 한 번에 재전송 + delivery row를 펼쳐 어떤 payload였는지 확인.

## 개발 범위

- **Track A — Run log live tail (SSE)**:
  - 새 store helper `GetRunLogPathAndStatus(runID)` — log path + run status를 한 번에. 또는 기존 `GetRun`을 사용.
  - 새 endpoint `GET /api/runs/{id}/log/stream?offset=N` — text/event-stream. 파일을 open + seek(offset) + 새 데이터 push. run이 terminal이면 현재 내용 push 후 stream 종료.
  - 클라이언트: IssueDetailPage의 Run history card 안에 "라이브 로그" 버튼 + 펼치면 stream을 받아 monospace 패널에 append.

- **Track B — Bulk webhook redelivery + payload preview**:
  - `Store.ResubmitWebhookDeliveryFailed(webhookID) (count int, err error)` — webhook의 모든 status='failed' 행을 'pending'으로 reset.
  - `POST /api/webhooks/{id}/deliveries/redeliver-failed`.
  - WorkspaceWebhookSection에 hook row에 "모두 재전송" 버튼 (`failed_delivery_count>0`일 때만).
  - delivery row 클릭으로 payload_json 펼치기 (formatted JSON view).

## 제외 범위

- Run log 양방향 stream (사용자가 명령 입력) — read-only.
- 다른 워크스페이스에 dead-letter 일괄 적용 — workspace 범위 X, single-webhook 범위.
- payload edit/replay — 그대로 재시도만.
