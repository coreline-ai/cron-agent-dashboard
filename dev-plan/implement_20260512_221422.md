## 개발 목적
- 코드 레벨 전문가 리뷰에서 지적된 High/Medium 항목 중 MVP 릴리스 전에 바로 해소 가능한 정합성 문제를 수정한다.
- 문서가 약속한 데이터 필드(`comment.truncated`, `autopilot_rule.next_run_at`)와 UI 동작(markdown 렌더)을 실제 구현과 일치시킨다.
- CLI runtime adapter가 codex/claude/gemini의 비대화형 실행 방식 차이를 반영하도록 보강한다.

## 개발 범위
- `comment.truncated` 컬럼 마이그레이션 및 store/query/write 경로 반영
- Autopilot `next_run_at` 재계산/DB 동기화
- `/api/system/vacuum` 실제 SQLite `VACUUM` 실행 및 반환값 개선
- `PUT /api/issues/{id}` body 필드의 JSON pointer 기반 처리
- `react-markdown` + `remark-gfm` 기반 안전 markdown 렌더링 도입(raw HTML 실행 금지)
- codex/claude/gemini runtime adapter CLI 인자 차이화 및 단위 테스트
- TODO/README/API/TRD 등 관련 문서 상태 동기화

## 제외 범위
- 실제 유료/로그인 CLI 계정으로 원격 모델 실행 E2E
