작성 일시: `2026-05-23 20:32:19 KST`

직전 사이클(`implement_20260523_201535.md`)에서 ① issue-scoped SSE를 in-process push로 전환했고 ② `/w/:slug/runs` 페이지를 추가했다. 두 후속을 잡는다.

## 개발 목적

1. workspace-level 페이지(`/w/:slug/runs`, `/w/:slug/chains`)도 push로 업데이트되도록 한다. 현재는 8초 폴링.
2. webhook dead-letter delivery를 운영자가 한 번에 재시도할 수 있게 한다. 현재 UI는 배지만 표시하고 액션이 없다.

## 개발 범위

- **Track A — workspace-scoped SSE**:
  - 기존 `IssueEventBus`에 workspace-scoped subscribe API 추가 (per-issue 구독은 그대로 유지).
  - `OnRunEvent(issueID, runID)`이 호출될 때 issueID로 workspace를 lazy lookup하고 workspace channel도 wake — 너무 비싸므로 캐시 (issueID → workspaceID map).
  - 새 SSE endpoint `GET /api/workspaces/{slug}/runs/stream`.
  - `WorkspaceRunsPage` + `WorkspaceChainsPage`가 EventSource로 구독해 query invalidation.

- **Track B — webhook redelivery**:
  - `Store.ResubmitWebhookDelivery(deliveryID)`: status='failed' 행을 status='pending' + next_attempt_at=now + error_message=''로 reset. 'delivered' 행은 ErrState.
  - `POST /api/webhooks/{id}/deliveries/{deliveryID}/redeliver`.
  - `WorkspaceWebhookSection`의 delivery list에 status='failed' row에 "재전송" 버튼.

## 제외 범위

- 새 store-level workspace event broker — 기존 IssueEventBus 확장으로 충분.
- 웹훅 일괄 재전송(모든 dead-letter) — 단일 row 재시도만.
