## 개발 목적

1. SSE 핸들러가 현재 1초 폴링으로 deltas를 fetch한다. 운영 자유도가 충분히 안정화된 시점에서 in-process notifier로 갈아치워 ① latency를 ~1s → ~ms로 떨어뜨리고 ② 이슈 상세 페이지가 열려 있어도 idle DB query가 0이 되도록 한다.
2. `/w/:slug/chains`는 chain 그룹 기준이라 개별 run 단위 운영(취소 / 로그 / agent 필터)이 한 번 더 클릭이 필요하다. workspace-wide **개별 run** feed 페이지(`/w/:slug/runs`)를 추가해 운영자가 한 화면에서 전체 워크스페이스의 run을 status/agent/chain로 필터하며 볼 수 있게 한다.

## 개발 범위

- **Track A — SSE in-process push**:
  - 새 `internal/store.RunEventNotifier` 인터페이스 + `Store.SetRunEventNotifier`.
  - `AppendRunEvent` 트랜잭션 commit 후 notifier 호출.
  - 새 `internal/app.IssueEventBus` (issueID → set of subscriber channels). `RunEventNotifier` 구현.
  - `serve()`가 EventBus 인스턴스를 만들고 store/httpapi에 주입.
  - SSE 핸들러는 1초 ticker 대신 EventBus 채널 + 60초 keep-alive timer로 전환. 백워드 호환: `?since=` 초기 catch-up은 그대로.
- **Track B — workspace runs feed**:
  - 새 페이지 `web/src/pages/WorkspaceRunsPage.tsx` at `/w/:slug/runs`.
  - 기존 `GET /api/workspaces/:slug/runs` 재사용 (Track G of 직전 plan에서 추가됨).
  - 필터: status (queued/running/done/failed/cancelled), agent (드롭다운), search (chain_id 또는 issue identifier substring).
  - 사이드바 메뉴 "run feed" 추가.
