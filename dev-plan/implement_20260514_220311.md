# implement_20260514_220311.md

작성 일시: `2026-05-14 22:03:11 KST`

이 문서는 직전 9차 보강 후 남은 후속 항목을 병렬 에이전트로 처리하기 위한 작업 범위와 검증 기준을 고정한다.

## 개발 목적
- 운영 안정성 잔여 항목인 stale active run cancel, worker panic 자동 차단을 보강한다.
- frontend 회귀 테스트 기반을 최소 구성으로 도입한다.
- run lifecycle/event label 중복을 단일 소스로 정리한다.

## 개발 범위
- Worker pool operational safety
  - active run heartbeat가 장시간 성공하지 못하면 해당 run context를 stale cancel reason으로 취소한다.
  - executor panic이 반복될 때 worker claim을 일정 시간 차단하는 circuit breaker를 추가한다.
- Frontend unit test foundation
  - Vitest + Testing Library 기반 최소 테스트 환경을 도입한다.
  - 핵심 순수/소형 컴포넌트 테스트를 추가한다.
- Frontend label single source
  - trigger / terminal reason / failure kind / cancel reason / run event label을 공용 모듈로 분리한다.
  - IssueDetailPage 인라인 label map을 제거한다.

## 제외 범위
- Git commit/push는 제외한다. 사용자가 별도 요청할 때 수행한다.
- OS PID 기반 강제 kill/chroot/sandbox는 제외한다. 이번 작업은 현재 worker process가 보유한 context cancel 기반 process-group termination 경로만 사용한다.
- 자동 chain 구현은 제외한다.
- 대형 UI 리디자인은 제외한다.
