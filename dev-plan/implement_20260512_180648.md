이 문서는 `Cron Agent Dashboard` 초기 스캐폴드 이후 남은 항목을 실행 가능한 순서로 재정리하고, 다음 병렬/순차 개발이 목적 밖으로 확장되지 않도록 하기 위한 후속 작업 문서다.

## 개발 목적

- 현재 구현된 foundation scaffold를 실제 사용 가능한 로컬 MVP로 끌어올린다.
- 남은 작업을 **blocking 순서**와 **병렬 가능 단위**로 분리한다.
- worker/scheduler/frontend/embed/release 작업이 서로 충돌하지 않도록 경계를 명확히 한다.
- 각 단계마다 자체 테스트를 정의해 “빌드만 되는 상태”가 아니라 “동작 확인 가능한 상태”로 전진한다.

## 문서 기반 제약 사항

- `issue.status`는 사용자 의도만 저장한다: `open | done | cancelled`.
- `run.status`는 실행 상태만 저장한다: `queued | running | done | failed | cancelled`.
- `execution_status`는 API derived field로 유지한다.
- mention은 assignee를 바꾸지 않고 `run.agent_id`만 변경한다.
- MVP는 workspace당 running run 1개만 허용한다.
- stdout log file은 10MB cap, comment는 64KB cap을 지킨다.
- 외부 bind 시 token 미설정이면 부팅 거부한다.
- raw HTML markdown rendering은 허용하지 않는다.
