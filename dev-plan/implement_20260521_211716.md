## Phase 1. B4 — main agent 재진입을 chain_depth 카운트에서 제외

### 목표

- 같은 chain 내에서 main agent(workspace PM hub)가 재진입할 때 `chain_depth`를 증가시키지 않고 `max_depth` 가드에서도 제외해, 6 worker × 2(hub) ≈ 12 depth 같은 운영 조정 없이 RFP/hub-PM 패턴이 기본값으로 동작하게 한다. linear chain(worker→worker)은 기존 그대로 +1.

### 구현 태스크

- [x] `enqueueAutoChainMention`: agent resolve를 guards 호출 **이전**으로 옮겨 hub-PM aware guards가 동작하도록 흐름 재배열.
- [x] `checkAutoChainGuards(cfg, run, agent)` 시그니처 변경 — `mention string` 대신 resolved `Agent` 인자. `!agent.IsMain && run.ChainDepth >= maxDepth`일 때만 depth-limit 차단.
- [x] `dispatchAutoChainRun`: `depth := run.ChainDepth + 1` 분기 — main 재진입은 부모 depth 그대로(`run.ChainDepth`), worker dispatch는 +1.
- [x] daily run/cost guard, `existingQueued` 가드, hub-PM duplicate exemption(`294fe2e`)는 그대로 유지.
- [x] `TestAutoChainMainAgentRevisitAllowed`: 재진입 후 `chain_depth == 1`(이전 `2`)로 갱신.
- [x] `TestAutoChainMainAgentRevisitStopsAtMaxDepth` 삭제 → `TestAutoChainHubPMOnlyWorkerDispatchesCountTowardMaxDepth`로 교체. Lead→W1→Lead→W2→Lead→W3 시나리오로 worker dispatch만 max_depth 카운트에 들어감을 검증.
- [x] 신규 `TestAutoChainHubPMFiveWorkersFitWithinMaxDepthFive`: 5 worker hub-PM 워크플로우(Sales→Lead→Planner→Lead→…→QA)가 `max_depth=5`에 정확히 들어가는지 검증(워커 5명 × Lead 재진입 4번).

### 자체 테스트

- [x] `go test -race ./internal/store/...` 통과
- [x] `go test -race ./...` 클린 (10 패키지)
- [x] 갱신된 main 재진입 테스트가 새 규칙에 부합. linear worker chain은 기존 동작 유지(`TestAutoChainHubPMOnlyWorkerDispatchesCountTowardMaxDepth`가 가드 통과/차단 모두 검증).

### 이슈 및 수정
