### 구현 태스크

- [x] Phase C.1 — TDD: `worktree_git_test.go` — 임시 git repo init → AllocateRunWorktree(git mode) → `.git` regular file 확인 → seed.txt 노출 확인 → `git worktree list` 등재 → cleanup → registry 제거 확인. git 미설치 환경은 `t.Skip()`.

## 잔여 리스크 / 후속 과제 (본 plan 범위 밖)

- codex `--json` 이벤트 스키마는 codex CLI 버전에 따라 변동 가능. fixture 기반 파서이므로 alpha-stage 키 추가/삭제에는 약함. 향후 codex CLI 메이저 업데이트 시 fixture 갱신 필요.
- import/export v2 history는 attachment 바이너리 자체를 동봉하지 않는다 (직렬화 크기 폭발). 별도 ZIP-bundle 옵션은 후속.
- git worktree 통합은 detached HEAD 또는 새 브랜치 정책을 강제하지 않는다. branch isolation 강화는 후속.
- Homebrew tap 등록은 사용자가 own-repo에서 진행해야 함 (`coreline-ai/homebrew-tap` 등).
