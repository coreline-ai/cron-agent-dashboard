## 개발 목적

backlog 마지막 항목 — **Auto-chain 고급 UI 후속**. RFP / hub-PM 워크플로우에서 한 chain이 8단계까지 굴러갈 수 있는데 (`implement_20260520_230031.md` e2e 사례), 운영자가 IssueDetail에서 chain 단위로 흐름을 한눈에 보기 어려웠다. 본 plan은 **같은 chain_id를 가진 run들을 그룹핑**해 시작/종료/depth/cost/token/마지막 status를 요약하는 ChainSummaryPanel을 IssueDetail에 추가한다.

## 개발 범위

- 같은 `chain_id`의 run들을 묶어 `ChainSummary` 데이터 구조로 집계.
- 새 컴포넌트 `ChainSummaryPanel` — 이슈의 모든 chain을 나열, 각 chain의 시작/마지막 활동/run 수/최고 chain_depth/총 cost/총 token/마지막 status 표시.
- 기존 IssueFlowGraph + Run History 위에 한 단으로 배치.
- vitest 1-2건으로 chain 그룹 핑 정확성 명세.

## 제외 범위

- chain 단위 운영 액션(전체 cancel, last-failed step retry) — 본 plan은 가시화만. 액션은 별도 plan.
- chain tree 편집 / stage-level resume — UI/UX 디자인이 더 큰 작업.
- 새 API endpoint — 본 plan은 기존 run list payload만 사용.
- workspace 차원 chain dashboard — 본 plan은 issue 안에서만.
