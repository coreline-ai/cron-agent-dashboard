# 구현 계획 — post_release_plan_v0.1_to_v0.2 기반 v0.1.x 안정화/사용성 개선

> 생성: 2026-05-14 23:04 KST  
> 기준 문서: `dev-plan/post_release_plan_v0.1_to_v0.2.md`  
> 목표: v0.2 전체 범위가 아니라, daily-use 시 즉시 체감되는 안전한 v0.1.x 개선을 먼저 구현한다.

---

## 1. 분석 요약

`post_release_plan_v0.1_to_v0.2.md`는 v0.1 릴리스 후 1~2주 동안 가장 답답할 영역으로 다음을 꼽는다.

| 우선 | 항목 | 즉시 구현 판단 |
|---|---|---|
| P1 | 명시 refresh 버튼 + Tab focus refetch | ✅ 낮은 리스크, 즉시 구현 |
| P1 | Agent name 자동완성 | ✅ 댓글 멘션 오타 방지, 즉시 구현 |
| P1 | Token/cost 추적 | ⏸️ CLI metrics 포맷 검증 필요. 설계 후 보류 |
| P2 | Frontend 단위 테스트 | ✅ 이미 기반 존재. 신규 컴포넌트 테스트 추가 |
| P2 | Autopilot snooze_until | ✅ 백엔드/프런트 작은 스키마 변경으로 구현 가능 |
| P2 | stale zombie 처리 | ✅ 현재 안정화 패치에 포함해 보강 |

이번 개발은 **측정/UX 개선 + Autopilot 일시정지 + 프로세스 추적 안전성**까지를 v0.1.x 범위로 묶고, token/cost는 실제 CLI 출력 포맷 확인 후 별도 v0.2 설계로 분리한다.

## 4. 완료 기준

- daily-use에서 사용자가 즉시 데이터를 새로고침할 수 있다.
- 댓글 작성 시 agent mention 오타 가능성이 줄어든다.
- Autopilot rule을 임시로 snooze하고, snooze 기간 동안 이슈가 자동 생성되지 않는다.
- startup self-check가 오래된 process group metadata를 무리하게 kill하지 않는다.
- 전체 테스트/빌드가 통과한다.
