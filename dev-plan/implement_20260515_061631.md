## 개발 목적

멀티 에이전트 안정화 리뷰에서 지적된 자원 관리/복구성/멘션 안정성 갭을 v0.1.x 범위에서 낮은 리스크로 보강한다. 현재 main은 리뷰 문서의 기준(`369454a`)보다 최신(`7262a5f`)이므로, 이미 반영된 post-release polish와 중복되지 않는 항목만 진행한다.

## 개발 범위

- CR-A: token/cost 추적의 **기반** 구현
  - run token/cost/model 컬럼 추가
  - runtime별 metrics parser 계약 추가
  - stdout/stderr의 JSON usage/cost 패턴을 best-effort로 파싱
  - run API/UI와 settings summary에 사용량 표시
- CR-B: run timeout 사용자 정의의 **기반** 구현
  - workspace 기본 timeout, agent/issue override 컬럼 추가
  - worker 실행 시 `issue > agent > workspace > executor default` 순서로 timeout resolve
- CR-C: 자동 retry의 **최소 안전 구현**
  - run attempt/max_attempts/next_retry_at 컬럼 추가
  - agent retry_policy_json 기반 max attempts 반영
  - timeout/executor_error만 backoff retry, exit_nonzero/worker_panic은 retry하지 않음
- HG-6: 다국어 agent mention regex 보강
  - Go regex를 Unicode letter/number 기반으로 변경
  - store/worker 회귀 테스트 추가

## 제외 범위

- 실제 CLI에 metrics 출력 옵션을 강제 추가하지 않는다. CLI별 출력 포맷은 버전별 차이가 있어 이번 범위는 best-effort parser와 저장 경로까지로 제한한다.
- 가격표 기반 비용 계산 UI는 제외한다. CLI가 cost 값을 제공하면 `total_cost_micros`에 저장한다.
- timeout/retry 설정 UI 전체 편집 화면은 제외한다. API/DB/worker 기반을 먼저 만든다.
- sub-issue, auto-chain, sandbox, multi-user/RBAC는 제외한다.
