## Phase 2. Gemini prompt argv → stdin/임시파일 전환

### 목표

- prompt 본문이 process argv(`/proc/PID/cmdline`)에 더 이상 노출되지 않는다. 자식 prompt 채널을 codex/claude와 동일한 stdin pipe로 통일하거나, stdin 미지원이면 임시파일 + `--prompt-file` 형태로 우회.

### 구현 태스크

- [x] `gemini --help` 확인 — `-p, --prompt` flag가 헤드리스 모드 트리거 + argv 텍스트가 "Appended to input on stdin (if any)"임을 확인. 즉 argv `-p ""` + stdin 본문이면 stdin이 곧 prompt.
- [x] `internal/worker/runtime/gemini.go::BuildCommand`를 `args := []string{"-p", ""}` + `commandWithPrompt`로 교체. prompt 본문은 stdin pipe(`bytes.NewReader`)로 전달.
- [x] `internal/worker/runtime/adapter_test.go`에서 기존 `TestGeminiAdapterBuildCommand`를 `TestGeminiAdapterBuildCommandPassesPromptViaStdin`으로 교체 — argv에 prompt body가 새지 않고, `-p` 또는 `--prompt` flag가 있고, model flag가 보존되며, stdin이 정확히 prompt body임을 검증.
- [x] `ParseMetrics`는 변경 없이 그대로(stderr/stdout 분리는 sanitize 패치에서 처리됨).
