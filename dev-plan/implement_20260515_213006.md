## 개발 목적
- 이미 저장되는 sub-issue/run lineage 데이터를 UI에서 추적 가능하게 만들고, auto-chain의 비용/횟수 가드를 workspace별로 설정 가능하게 한다.
- 비용 사용량 확인과 export/import CLI alias를 추가해 운영성을 높인다.
- MCP/sandbox/RBAC처럼 설계·보안 비용이 큰 항목은 장기 후보로 유지한다.

## 개발 범위
- React Flow 기반 Issue/Sub-issue/Run lineage graph 기본형
- `/api/usage/summary` 기반 7일/30일 cost dashboard 보강
- workspace별 auto-chain max depth / daily run limit / daily cost limit / dry-run guard
- `export`/`import` CLI alias 추가
- 관련 문서와 테스트 보강

## 제외 범위
- MCP server 실제 구현
- Sandbox/RBAC
- Auto-chain workflow builder
- 모델별 단가 입력/가격표 관리
- 외부 queue 교체
