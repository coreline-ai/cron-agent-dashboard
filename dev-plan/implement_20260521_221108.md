## 개발 목적

남은 backlog 7개 항목 중 첫 사이클: **C2 — claude `--print` adapter hang 정공 fix**. 운영 우회로 Lead runtime을 codex로만 유지 중인 상태에서 claude runtime을 다시 안전하게 쓸 수 있게 한다.

직전 spike 결과:

| 입력 방식 | 결과 |
|---|---|
| `echo "..." \| claude -p --input-format text` | ✅ 정상 응답(text) |
| `claude -p "..."` (positional argv) | ✅ 정상 응답(argv) |
| `echo "..." \| claude --print` (직전 adapter 호출 형태) | ❌ stdin이 prompt로 인식되지 않아 interactive mode로 진입 → hang |

즉 `--input-format text` 명시가 fix. argv 사용은 Gemini와 동일한 `/proc/<pid>/cmdline` 노출 risk이므로 채택하지 않는다.

## 개발 범위

- `internal/worker/runtime/claude.go::BuildCommand`에 `"--input-format", "text"` 인자 추가. prompt 본문은 그대로 `commandWithPrompt`(stdin pipe)로 전달.
- `internal/worker/runtime/adapter_test.go`에서 ClaudeAdapter 케이스 갱신 — args 시퀀스에 `--input-format text` 포함, stdin이 prompt 본문이 되는 계약 명세.
- `internal/worker/runtime/sanitize.go` 또는 stdout sanitize는 그대로(이미 runtime-agnostic).
- (옵션) dev 환경에서 RFP-style 워크스페이스로 Lead runtime을 claude로 되돌려 e2e 검증.
- 문서 — 직전 dev-plan(`implement_20260520_230031.md`)의 잔여 리스크 섹션과 TODO.md의 후속 항목을 closed로 표시.
