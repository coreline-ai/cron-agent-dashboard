## 개발 목적

자체 전문가 코드 리뷰에서 P1/P2로 확인된 실행 취소 및 상태 전이 문제를 보강한다. 개인용 AI 에이전트 작업 트래커에서 queued/running 작업을 사용자가 예측 가능하게 취소하고, 취소 race로 이미 취소한 작업이 외부 CLI에서 계속 실행되는 가능성을 줄인다.

## 개발 범위

- queued/running run 취소 API/Store 정책 통일
- worker pool cancel race 보강
- issue done/cancel/delete, workspace delete의 active run 체크 원자성 개선
- 이슈 생성 API 응답에 생성 run 포함
- 관련 프론트엔드 버튼 활성 조건 및 문구 정합성 개선
- API/UX 문서 업데이트
- 회귀 테스트 추가 및 전체 검증

## 제외 범위

- issueSelectBase N+1 구조 리팩터링
- 취소된 run의 부분 stdout_path 영구 회수 정책

- 대규모 UI 리디자인
- Release 태그 생성/배포

## Phase 2. Worker cancel race 보강

### 목표

HTTP cancel 요청이 worker의 cancel function 등록보다 먼저 들어와도, 등록 직후 즉시 취소 신호가 적용되도록 한다.

### 구현 태스크

- [x] worker pool에 pending cancel set 추가
- [x] `CancelRun`이 active cancel func 없을 때 pending cancel로 기록
- [x] claim 직후 pending cancel 확인 및 즉시 context cancel
- [x] worker pool 단위 테스트 추가
