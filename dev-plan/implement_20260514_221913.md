이 문서는 v0.1 릴리스 전후 우선순위 항목을 병렬 에이전트로 구현하고, 최종 release tag까지 진행하기 위한 작업 문서다.

## 개발 목적
- v0.1.0 릴리스에 필요한 문서/릴리스 노트/빌드 검증을 정리한다.
- 남은 운영 안정성 항목인 재시작 후 orphan process group 정리를 보수적으로 구현한다.
- 구현 완료 후 main 커밋/푸시 및 v0.1.0 tag push로 GitHub Release workflow를 트리거한다.

## 개발 범위
- P1. 릴리스 문서
  - `CHANGELOG.md` 추가
  - `docs/RELEASE_NOTES_v0.1.0.md` 추가
  - daily-use 운영 체크리스트 문서 추가 또는 기존 문서 반영
- P2. 재시작 후 orphan process group 정리
  - run process pid/pgid 기록용 migration 추가
  - executor process start 후 pid/pgid best-effort 기록
  - startup self-check에서 DB상 running orphan process group을 보수적으로 종료 후 orphan run recovery 수행
  - Unix 구현 + non-Unix safe no-op
- P3. 릴리스 검증/태그
  - `make check`
  - `make release-build VERSION=v0.1.0`
  - `git tag v0.1.0` 및 push

## 제외 범위
- 자동 chain 구현은 제외한다.
- OS sandbox/chroot/bwrap 권한 격리는 제외한다.
- GitHub Actions 실행 결과 모니터링 자동화는 제외한다. tag push 트리거까지만 수행한다.
- 이미 main에 반영된 `5fb915c`를 되돌리거나 커밋 분할하지 않는다.
