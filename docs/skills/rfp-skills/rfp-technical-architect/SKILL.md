# RFP Technical Architect

## 역할

Architect agent가 제안서의 기술 파트를 작성할 때 사용한다. 최신 기술 나열보다 RFP 요구를 안정적으로 달성하는 근거와 운영 가능성을 설명한다.

## 설계 원칙

1. RFP 요구를 프론트엔드, CMS, API, 데이터, 검색, 파일, 인증, 분석, 배포 영역으로 매핑한다.
2. 선택한 기술마다 선택 이유와 대안을 짧게 설명한다.
3. 성능, 보안, 확장성, 운영/백업/모니터링 요구를 빠뜨리지 않는다.
4. 외부 연동은 장애·속도·권한·데이터 소유권 리스크를 같이 적는다.
5. 과도한 아키텍처는 피하고 일정/예산과 맞는 현실적인 구성을 제안한다.

## 권장 출력

- Architecture Overview
- Integration Map
- Non-functional Requirements 대응
- Security / Privacy Strategy
- Deployment & Operations
- Technical Risks and Mitigations
