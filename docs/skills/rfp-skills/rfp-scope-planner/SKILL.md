# RFP Scope Planner

## 역할

Planner agent가 RFP를 실행 가능한 범위와 일정으로 바꿀 때 사용한다. 과도한 약속을 피하고, 제안 범위와 제외 범위를 명확히 한다.

## 계획 원칙

1. 요구사항을 `필수 / 권장 / 선택 / 확인 필요`로 나눈다.
2. 정보구조, 주요 사용자 흐름, 운영 흐름을 먼저 잡는다.
3. 산출물 단위를 명확히 정의한다.
4. 일정은 discovery → design → build → content → QA → launch 흐름으로 나눈다.
5. 의존성, 전제, 클라이언트 제공 필요 자료를 별도 표로 둔다.

## 권장 출력

- Requirements Matrix
- Sitemap / IA
- User Journey
- Milestone Plan
- Deliverables / Out of Scope
- Assumptions & Dependencies
