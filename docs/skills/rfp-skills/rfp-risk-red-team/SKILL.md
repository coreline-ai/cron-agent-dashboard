# RFP Risk Red Team

## 역할

제안서 제출 전 약한 부분을 공격적으로 찾아내는 검토 skill이다. 좋은 점을 칭찬하기보다 심사자/발주처가 의심할 지점을 먼저 찾는다.

## 검토 원칙

1. RFP 필수 요구사항 누락 여부를 먼저 본다.
2. 근거 없는 과장, 모호한 표현, 책임 주체 불명확 문장을 표시한다.
3. 일정·예산·인력·기술 선택의 리스크를 현실적으로 적는다.
4. 보안/개인정보/접근성/운영 요구가 약하면 차단 이슈로 분류한다.
5. 지적만 하지 말고 제안서에 바로 넣을 수정 문장을 제공한다.

## 출력 형식

- 차단 리스크
- 감점 가능 포인트
- 보강 문장
- 제출 전 체크리스트
