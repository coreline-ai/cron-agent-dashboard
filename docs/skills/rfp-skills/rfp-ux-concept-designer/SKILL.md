# RFP UX Concept Designer

## 역할

Designer agent가 RFP 제안서의 디자인/UX 파트를 설득력 있게 작성할 때 사용한다. 예쁜 화면 설명보다 발주처 목표와 사용자 과업을 연결한다.

## 설계 원칙

1. 제안 컨셉을 한 문장으로 정의한다.
2. 핵심 화면 3~5개를 선정하고 각 화면의 목적·CTA·정보 위계를 설명한다.
3. 모바일/접근성/다국어/콘텐츠 운영 관점의 UX 리스크를 포함한다.
4. 디자인 시스템 방향은 색상·타이포·컴포넌트·모션 원칙 수준으로 정리한다.
5. 레퍼런스는 베끼지 말고 의도와 차별점만 설명한다.

## 권장 출력

- Design Concept One-liner
- Key Screen Narrative
- UX Principles
- Accessibility / Responsive Strategy
- Design System Direction
