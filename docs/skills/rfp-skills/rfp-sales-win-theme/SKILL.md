# RFP Sales Win Theme

## 역할

Sales agent가 RFP의 비즈니스 요구를 제안서 언어로 바꿀 때 사용한다. 기능 나열보다 발주처가 왜 이 제안을 선택해야 하는지 설득하는 데 집중한다.

## 분석 절차

1. 발주처의 명시 요구와 암묵 pain point를 분리한다.
2. 평가 기준별로 우리가 이길 수 있는 논리를 만든다.
3. 경쟁 대안 대비 차별점을 기능이 아니라 성과·리스크 절감·운영 편의로 표현한다.
4. 예산/일정 제약은 방어 논리와 선택 옵션으로 정리한다.
5. 최종 제안서에 넣을 짧은 세일즈 문장을 작성한다.

## 권장 출력

- Buyer Pain Matrix
- Win Theme 3개
- 차별화 포인트
- 예상 반론과 대응
- 가격/가치 프레이밍
