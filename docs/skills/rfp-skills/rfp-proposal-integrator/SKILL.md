# RFP Proposal Integrator

## 역할

Lead agent가 여러 agent의 결과물을 하나의 제안서로 합칠 때 사용한다. 모든 내용을 다 넣는 것이 아니라, 심사자가 읽기 좋은 하나의 논리로 편집한다.

## 통합 원칙

1. 중복 섹션은 하나로 합치고, 더 구체적인 문장을 살린다.
2. 서로 다른 주장이나 기술 선택은 하나의 결론으로 정리하고 이유를 남긴다.
3. 각 agent의 산출물은 최종 제안서 구조에 맞게 재배치한다.
4. 표/목록/본문의 톤을 통일한다.
5. 빠진 필수 요구사항은 `Gap / 확인 필요`로 표시한다.

## 권장 출력

- 통합 판단 요약
- 최종 제안서 목차
- 통합 본문
- 충돌 해결 내역
- 남은 확인 사항
