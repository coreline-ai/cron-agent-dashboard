### 3.3 `/w/:slug/issues/:id` — 이슈 상세

```
┌──────────────────────────────────────┬──────────────────────┐
│ [← 보드로]                            │ 상태:    🔵 진행 중   │
│                                      │ 담당:    NewsLead    │
│ NEWS-15                              │ 만든이:   사용자       │
│ 오늘 뉴스 정리                          │ 생성:    2분 전       │
│                                      │                      │
│ Reddit r/MachineLearning 상위 5개로... │ ─── 작업 ───────────  │
│                                      │ [재실행]              │
│ ─── 댓글 ─────────────────────────────  │ [취소]                │
│                                      │ [편집]                │
│ ⚙ 시스템                              │ [삭제]                │
│ 2분 전                                │                      │
│ NewsLead 실행을 시작했습니다              │ ─── 마지막 오류 ────  │
│ ────────────────────────────         │ (없음)                │
│                                      │                      │
│ 🤖 NewsLead                          │ ─── Run 이력 ───────  │
│ 30초 전 · run #1                      │ run #1               │
│ ────────────────────────────         │ 진행 중               │
│ # 오늘의 AI 뉴스                       │ 30초 경과             │
│ | 제목 | 링크 | 점수 |                  │ [로그 보기]           │
│ |--|--|--|                            │                      │
│ ...                                  │                      │
│ [로그 다운로드]                        │                      │
│                                      │                      │
│ 👤 사용자                             │                      │
│ 1분 전                               │                      │
│ ────────────────────────────         │                      │
│ @Writer 이걸로 블로그 글 써줘            │                      │
│                                      │                      │
│ ⚙ 시스템                              │                      │
│ 1분 전                                │                      │
│ Writer 실행을 시작했습니다 (run #2)       │                      │
│ ────────────────────────────         │                      │
│                                      │                      │
│ ┌─────────────────────────────────┐   │                      │
│ │ 댓글 작성 (@AgentName 으로 위임)    │   │                      │
│ │                                 │   │                      │
│ │ [작성]                          │   │                      │
│ └─────────────────────────────────┘   │                      │
└──────────────────────────────────────┴──────────────────────┘
```

**사이드바 항목 (상세)**:
- **이슈 상태** (issue.status): open / done / cancelled
- **실행 상태** (execution_status): running / queued / done / failed / cancelled / idle  ← 보드 배지 동일
- **담당** (assignee_agent): NewsLead — 멘션이 들어와도 안 바뀜
- **최근 실행** (last_run_agent): Writer — 마지막 run의 agent. [재실행] 대상.
- 만든이: 사용자 / 자동화
- 생성: 2분 전
- **마지막 오류** (run.error_message): execution_status가 `failed` 또는 `cancelled`일 때만 표시
- (Phase 2) 부모 이슈 / 서브 이슈 트리

**상태별 사이드바 변형**:
- `running` / `queued`: "마지막 오류" 섹션 숨김
- `failed` / `cancelled`: **마지막 오류 메시지** + 빨강/회색 강조
- `done`: "마지막 오류" 섹션 숨김, "최근 결과" 미니 미리보기 옵션

**좌측 (본문 영역)**:
- 헤더: identifier + 제목 + 본문 (markdown 렌더, **raw HTML 금지**)
- 댓글 스레드 (시간순, 위→아래)
  - 에이전트 댓글: 🤖 아이콘 + 에이전트 이름 + 시각 + run 번호 + markdown content (raw HTML 금지)
    - `truncated=true` 인 경우: "전체 로그는 [로그 보기](...) (총 X KB)" 링크
  - 사용자 댓글: 👤 아이콘 + 시각 + 본문
  - 시스템 댓글: ⚙ 아이콘 + 시각 + 본문 (회색). 발생 케이스:
    - "<Agent> 실행을 시작했습니다 (run #N)"
    - "사용자가 실행을 취소했습니다"
    - "이슈가 취소되었습니다"
    - "에이전트 @<Name>을 찾을 수 없습니다" (멘션 매칭 실패)
    - "멘션 둘 이상 — 첫 번째(@<Name>)만 적용됩니다"
    - "이미 @<Name>에게 대기 중인 작업이 있습니다" (중복 dispatch)
    - "재시작 중 진행 작업이 취소되었습니다" (orphan 정리)
- 하단 댓글 입력 폼 (markdown, `@AgentName` 자동완성)
  - **힌트 텍스트**: "@에이전트 멘션은 같은 이슈에 새 실행을 추가합니다 (담당자는 바뀌지 않음)"
  - **체이닝 정책 안내**: "현재는 사용자 댓글의 명시 멘션만 실행됩니다. agent 결과 댓글의 @멘션은 자동 실행되지 않습니다."

**우측 사이드바 (요약)**:
- 이슈 상태 / 실행 상태 / 담당 / 최근 실행 / 만든이 / 생성 시각
- (Phase 2 후보) 부모 이슈 / 서브 이슈 트리
- **마지막 오류 메시지** (execution_status가 failed/cancelled일 때만 표시)
- 작업 버튼:
  - [재실행]: execution_status가 done/failed/cancelled/idle일 때만 활성. 대상 agent는 **마지막 run의 agent**.
  - [대기 취소/실행 취소]: execution_status가 queued/running일 때 활성. active run 한 개만 cancel하고 issue.status는 open 유지.
  - [완료로 표시]: issue.status가 open이고 execution_status가 done/failed/cancelled/idle일 때 활성. PUT status='done'.
  - [이슈 취소]: issue.status가 open이고 execution_status가 queued/running이 아닐 때 활성. PUT status='cancelled' (queued run은 백엔드에서 자동 취소하지만 UI는 실행 취소와 구분).
  - [편집]: 항상 활성.
  - [삭제]: execution_status가 queued/running이 아닐 때.
- Run 이력 (최신 위, 접힘 가능). 각 run의 status 배지(`queued/running/done/failed/cancelled`) + trigger_type 라벨 (생성/멘션/자동/재실행)

**스크롤/포커스 정책**:
- 사용자가 맨 아래 보고 있으면(스레드 끝 근처) 새 댓글 도착 시 자동 스크롤
- 위쪽을 보고 있으면 우측 하단에 "새 댓글 N개" 토스트, 클릭 시 이동

**Polling**:
- `execution_status` ∈ {`queued`, `running`}일 때 3초 간격 댓글 + runs fetch
- 그 외는 정지 (issue.status='done'이라도 새 멘션으로 execution_status가 'queued'로 바뀌면 polling 재개)

```
│ 어제 24시간 동안의 핫이슈 5개를 정리해줘   │
└────────────────────────────────────────┘

담당:        [NewsLead (메인) ▼]

            [2026-05-20 09:00 직접 입력]

활성화:      [● ON  ○ OFF]

           [취소]  [저장]
```

- preset 선택 시 cron_expr 자동 생성, 직접 입력 모드로 전환 가능
- 다음 실행 시각이 룰 카드에 표시 → 직관적
- `snooze_until`이 미래면 룰은 ON 상태여도 "일시정지" 배지로 표시하고, 만료 전 trigger는 no-op 처리

## 4. 인터랙션 패턴

### 4.1 상태 배지 색상

**execution_status (보드 카드 + 사이드바)**:
| Status | 색상 | 설명 |
|---|---|---|
| queued | gray-400 | 큐에 있고 워커 pickup 대기 |
| running | blue-500 (pulsing) | 에이전트 실행 진행 중 |
| done | green-500 | 마지막 run 정상 종료 (exit 0) |
| failed | red-500 | 마지막 run 실패 (CLI 오류/타임아웃/exit≠0) |
| cancelled | amber-500 | 마지막 run 취소됨 (사용자/orphan/shutdown) |
| idle | gray-300 | run이 한 번도 없음 (예외적) |

**issue.status (보드 탭 / 사이드바)**:
| Status | 표현 | 설명 |
|---|---|---|
| open | 기본 (배지 없음) | 진행 중이거나 진행 가능 |
| done | 초록 체크 | 사용자 또는 자동 완료 |
| cancelled | 회색 X | 사용자가 이슈 자체 취소 |

### 4.2 Toast 알림
- 이슈 생성 직후: "이슈 NEWS-15 생성됨" (3초)
- 댓글 작성 후 멘션 dispatch: "@Writer에게 작업이 전달되었습니다" (3초)
- 멘션 둘 이상: "@Writer만 적용됩니다 (첫 번째)" (4초, 정보색)
- 멘션 실패: "에이전트 @Foo를 찾을 수 없습니다" (5초, 경고색)
- 취소 성공: "실행을 취소했습니다" (3초)
- API 에러 (500): "잠시 후 다시 시도해주세요" (5초, 빨강)

### 4.3 키보드 단축키 (Phase 2 후보)
- `c` → 새 이슈 모달
- `g b` → 보드
- `g a` → 에이전트
- `g p` → autopilot

### 4.4 체이닝 정책 표시

현재 UX는 **explicit-first + workspace opt-in auto-chain**을 전제로 한다.

- 댓글 입력에서 사용자가 직접 작성한 `@AgentName` 명시 멘션은 새 run을 만든다.
- agent 결과 댓글 안의 `@AgentName`은 기본적으로 텍스트로만 표시된다.
- `/settings`에서 workspace `agent 결과 @mention 자동 체이닝 허용`을 켠 경우에만 depth/run/cost/dry-run guard를 통과해 자동 dispatch된다.
- 이유: 무한 루프, 비용 폭주, hallucinated mention 실행을 기본 차단하고 필요한 workspace에서만 명시적으로 허용하기 위해서다.
- 자동 dispatch/skip 결과는 system comment와 run event timeline에 남긴다.
