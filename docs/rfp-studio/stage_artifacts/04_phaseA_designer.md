## 3. 타이포그래피·아이콘·그리드 가이드

### 타이포그래피

| 용도 | 한국어 | 영문/중문 | Weight |
|---|---|---|---|
| **H1 (히어로)** | Pretendard | Inter | Bold 700 / 40px (D) · 28px (M) |
| **H2 (섹션 제목)** | Pretendard | Inter | SemiBold 600 / 32px (D) · 24px (M) |
| **H3 (카드 제목)** | Pretendard | Inter | Medium 500 / 22px (D) · 18px (M) |
| **Body** | Pretendard | Inter | Regular 400 / 16px (D) · 15px (M) |
| **Caption / Label** | Pretendard | Inter | Regular 400 / 13px |
| **중문 본문** | Noto Sans SC | — | Regular 400 / 16px |

> **선정 이유**: Pretendard + Inter — 한글·라틴 호환성 우수, 오픈소스, Variable Font로 성능 최적화 가능. 중문은 Noto Sans SC로 보완.

### 아이콘 스타일

- **라이브러리**: Lucide Icons (오픈소스, MIT)
- **스타일**: Outlined, Stroke 1.5px, 24×24 기본 / 20×20 소형
- **컬러**: Primary Dark 단색 기본, 활성 상태 시 Primary Blue
- **제품 카테고리 아이콘**: 커스텀 SVG 라인 아이콘 별도 제작 (컨트롤러·센서 등 주요 5종)

### 그리드 시스템

| Breakpoint | 컬럼 | Gutter | Margin |
|---|---|---|---|
| Desktop (≥1280px) | 12col | 24px | 80px (좌우) |
| Tablet (768–1279px) | 8col | 20px | 40px |
| Mobile (<768px) | 4col | 16px | 20px |

- **최대 콘텐츠 폭**: 1200px (센터 정렬)
- **섹션 간 수직 여백**: 80px (D) / 60px (T) / 48px (M)

## 5. 반응형 전략

### Breakpoint 정의

| 구분 | 범위 | 기준 |
|---|---|---|
| **Mobile** | < 768px | **설계 기준 (Mobile First)** |
| **Tablet** | 768px – 1279px | 중간 확장 |
| **Desktop** | ≥ 1280px | 풀 레이아웃 |

### 모바일 우선 — 우선 노출 컴포넌트 전략

| 화면 | 모바일 우선 노출 | 축소/숨김 |
|---|---|---|
| **메인** | 히어로(CTA 강조) → 제품 카테고리(2col 스와이프) → CTA 배너 | 신뢰 배지 바 → 아이콘만 남김, 뉴스 → 2개로 축소 |
| **제품 목록** | 필터 버튼(바텀시트) → 1col 카드 리스트 | 사이드 필터 → 바텀시트 전환, 비교 기능 → 하단 플로팅 바 |
| **제품 상세** | 이미지 스와이프 → 스펙 요약 → CTA 고정 바 | 2col → 1col 스택, 탭 유지 |
| **문의** | 폼 (1col 풀폭) → 연락처 하단 접힘 | 지도 → 접힘 아코디언, 연락처 → 아코디언 |
| **GNB** | 햄버거 메뉴 + 언어 전환 아이콘 유지 | 풀 메뉴 → 슬라이드 드로어 |

### 주요 반응형 패턴

- **네비게이션**: Desktop=풀 GNB · Tablet=축약 GNB · Mobile=햄버거 드로어
- **카드 그리드**: Desktop 4col → Tablet 2col → Mobile 1col (제품 목록은 Mobile 2col 유지)
- **테이블**: Desktop 풀 테이블 → Mobile 카드형 변환 (스펙 테이블)
- **이미지 갤러리**: Desktop 그리드 → Mobile 스와이프 캐러셀
- **폼**: 모든 breakpoint에서 1col 스택 (일관성)
- **CTA**: Mobile에서 하단 고정(sticky) — 제품 상세의 "문의하기"는 스크롤 시 하단 바로 전환
- **폰트 스케일**: Desktop 기준 대비 Mobile은 H1 70%, H2 75%, Body 94% 비율
